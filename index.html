Educational beacons

Android application that portrays the use of beacons in both secure and insecure settings.

View project on GitHub

Our application

We are working on a solution that will enable any university to reliably relay information to their students through an application. Once it detects a beacon, it will notify the user -- contextually relevant information will be a tap away.

The application can also be configured to work with a school's website; however, information on the website would need to be maintained in order for the app to stay relevant. This can be done through any web service. An example can be seen below:

The application also takes advantage of Eddystone-EID for two-factor authentication. This requires the user to be near a beacon transmitting Ephemeral Identifiers and successfully resolve it in order to become authorized to access a location.

Proposed features

  • Eddystone-URL support (insecure)

  • Configurable; just declare what URL the application should look for

  • Eddystone-EID support (secure)

  • Using Eddystone-EID, information can be catered to students based on location as a second layer of security

Development status

  • Eddystone-URL: Application can be configured with a URL distributed by a beacon, which will notify the user that they are near it with a notification.
  • Eddystone-EID: Using available hardware by Accent beacons, we were able to successfully resolve Eddystone-EID packets and authenticate users.
