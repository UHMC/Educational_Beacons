Educational beacons

Android application that portrays the use of beacons in both secure and insecure settings.

View project on GitHub

Introduction

In the ever increasing field of technology, new advancements are bound to occur. For the average person, contextual technology is becoming more and more impactful in their lives. Whether it's social media applications asking them to make a post in the event of a disaster so that their family knows they're okay or a simple post saying "I'm at the coffee shop!"

Our application

We have developed a solution that enables any university to reliably relay information to their students through an application. Once it detects a beacon, it will notify the user. The information will be a tap away.

The application can be configured with a school's website. Information on the website needs to be maintained in order for the app to stay relevant. This can be done through any web service. An example can be seen below:

Proposed features

  • Eddystone-URL support (insecure)

  • Configurable; just declare what URL the application should look after

  • Eddystone-EID support (secure)

  • Using Eddystone-EID, information can be catered to students based on location, adding a second layer of defense where physical location is necessary.

Development status

  • Eddystone-URL: Application can be configured with a URL distributed by a beacon, which will notify the user that they are near it with a notification.
  • Eddystone-EID: No available hardware, simulated beacons are not working using the Android Beacon Library. MySQL cannot be used without extra backend web development. Google Endpoints might work instead. There is a lack of support as it is a very recent development.
