Educational beacons

Android application that portrays the use of beacons in both secure and insecure settings.

View project on GitHub

Our application

Our application is a demonstration to show the two different scenarios where beacons are present. One scenario is that of an insecure setting. Eddystone-URL and Eddystone-UUID can be cloned and tracked and should be used in ways that affect the public. In our demonstration, we begin to look for a beacon stationed at a school's cafeteria. Because it's a public place, it doesn't matter if the beacon is cloned or tracked.

The application can also be configured to work with a school's website; however, information on the website would need to be maintained in order for the app to stay relevant. This can be done through any web service. An example can be seen below:

The demonstration also shows how beacons would work in a secure setting. Eddystone-EID can be used for two-factor authentication. This requires the user to be near a beacon transmitting Ephemeral Identifiers and successfully resolve it in order to become authorized to access a location.

This kind of location-based security would stop cyber attacks from outside of a location. For example, even if a hacker were to get my password for my account, Eddystone-EID can be implemented so that you need to be near a specific beacon in order to log in, which would be impossible for a hacker in another country.

Proposed features

  • Eddystone-URL support (insecure)

  • Configurable; just declare what URL the application should look for

  • Eddystone-EID support (secure)

  • Demonstrates a secondary, location-based authentication to a login.

Development status

  • Eddystone-URL: Application can be configured with a URL distributed by a beacon, when found, it will automatically go to the website.

  • Eddystone-EID: Using available hardware by Accent beacons, we were able to successfully resolve Eddystone-EID packets and authenticate users.
